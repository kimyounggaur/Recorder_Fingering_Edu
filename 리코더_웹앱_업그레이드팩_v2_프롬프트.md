# 1. ⚠️ 운지 데이터 정오표 — 반드시 먼저 읽을 것

외부 공식 운지표(Yamaha 소프라노 바로크 차트 등)로 재검증한 결과다.

1. **바로크식 파(F5) = `T0 L1 L2 L3 R4 R6 R7` (5번 구멍만 열기)** — 원본 마스터 프롬프트의 데이터가 **정확하다.**
   - ❌ 주의: 일부 다른 프롬프트 문서에는 바로크식 파가 `[1,1,1,1,0,1,1,1]`(**4번을 열고 5·6·7을 막는** 형태)로 잘못 적힌 버전이 유통되고 있다. 이는 오류이므로, 과거 문서의 JSON을 재사용할 때 반드시 이 항목을 원본 마스터 프롬프트 기준으로 맞춘다.
2. **독일식 파(F5) = `T0 L1 L2 L3 R4`** — 정확함.
3. **높은 레(D6) = `L2` 하나만** — 엄지(T0)까지 떼는 **유일한 기본음**. U1에서 추가한다.
4. 높은 도(C6) = `T0 L2` — 정확함.

테스트 코드에 아래 회귀 케이스를 추가해 이 오류의 재유입을 차단한다.

```ts
// fingerings.test.ts에 추가
it("바로크식 파는 5번 구멍만 연다", () => {
  expect(FINGERINGS.baroque.fa).toEqual(["T0", "L1", "L2", "L3", "R4", "R6", "R7"]);
  expect(FINGERINGS.baroque.fa).not.toContain("R5");
});
it("높은 레는 왼손 가운데손가락만 막는다", () => {
  expect(FINGERINGS.baroque.highRe).toEqual(["L2"]);
  expect(FINGERINGS.german.highRe).toEqual(["L2"]);
});
```

[테스트]

# U2 — 합성음 폴백 고품질화 (리코더다운 소리)

## 목표

샘플이 없어도 "리코더답게" 들리는 Web Audio 폴백을 명세대로 구현한다.

## 실행 프롬프트

```text
RecorderAudioEngine의 합성 폴백(WebAudioRecorderEngine)을 아래 명세로 교체해.
오디오 추상화 인터페이스와 자동재생 정책 처리는 기존 그대로 유지한다.

[주파수 테이블 — frequencies.ts 신설, 단일 관리]
C5 523.25 / D5 587.33 / E5 659.25 / F5 698.46 / G5 783.99
A5 880.00 / B5 987.77 / HIGH_C 1046.50 / HIGH_D 1174.66
(평균율 A4=440Hz 기준. audioKey와 1:1 매핑)

[음색 합성 그래프 — 리코더는 기음이 지배적인 플루트계 악기]
1. osc1: sine, 기음 f, gain 1.0
2. osc2: sine, 2f, gain 0.18
3. osc3: sine, 3f, gain 0.08
4. 어택 치프(chiff): 화이트노이즈 30ms를 bandpass(중심 f×2, Q≈8)로 걸러
   gain 0.12로 첫 30ms에만 섞는다. 리코더 특유의 '투' 하는 발음이 된다.
5. 마스터 엔벨로프: attack 40ms → sustain 0.8 → release 200ms (exponentialRamp)
6. 미세 표정: sustain 구간에 4.5Hz, ±2% 진폭 비브라토(LFO→gain) — 과하면 끄기 가능하게.

[운영 규칙]
1. 재생 길이 기본 700ms, 연습·퀴즈 정답 재생은 500ms 파라미터화.
2. 새 음 재생 시 이전 음은 80ms 페이드아웃 후 노드 disconnect — 누수 금지.
3. 모든 노드는 재생마다 생성/해제하되 AudioContext는 재사용.
4. README에 "임시 합성 음색이며 실제 리코더 녹음이 아님"을 명시(원본 §8.3 준수).

[검증]
- 9음을 1→9 순서로 연속 재생하는 데모 버튼(디버그 오버레이 안)으로 음정 상승 확인
- 30회 연타 후 크래시·잡음·메모리 증가 없음을 성능 탭으로 확인하고 결과 보고
```

## 완료 기준

- 9음이 모두 다른 높이로, '뚜' 하는 발음감과 함께 재생된다
- 연타 시 이전 음이 뒤늦게 울리지 않는다(원본 §7.4와 동일 기준)

# U3 — 기초 배우기 모듈 (자세·잡는 법·입 모양·텅잉·손가락 위치)

## 목표

운지 이전 단계의 기초 교육 콘텐츠를 독립 모듈로 추가한다. **문구는 아래 원문 그대로**, 그림은 원본 §1.4 금지사항에 따라 교과서 스캔이 아닌 자체 제작 SVG 일러스트로 만든다.

## 실행 프롬프트

```text
features/basics 모듈을 신설해 "기초 배우기" 화면을 만들어. 헤더에서 진입하는
독립 화면이며, 카드 슬라이드(터치 스와이프 + 이전/다음 버튼 + 진행 점 1/5~5/5)로 구성한다.
아래 교육 문구는 한 글자도 바꾸지 말고 그대로 사용한다.

[카드 1. 바른 자세] — 카드 내부 [앉아서|서서] 세그먼트 전환
■ 앉아서
- 의자 등받이에서 허리를 떼고 앉습니다.
- 두 발을 어깨너비 정도로 벌려 바닥에 댑니다.
- 발은 바닥에 가볍게 붙여 중심을 잡도록 합니다.
- 손가락이나 어깨의 긴장을 풀어 줍니다.
■ 서서
- 양발을 어깨너비 정도로 벌려서 몸의 중심을 잡도록 합니다.
- 머리를 똑바로 하고 시선은 정면을 향합니다.
- 허리를 펴고 팔꿈치는 몸에서 조금 떨어지게 합니다.
- 손가락이나 어깨에 힘이 들어가지 않도록 긴장을 풀어 줍니다.
일러스트: 의자에 앉은/서 있는 사람의 단순 라인 SVG. 허리·발·어깨 포인트에 점 표시.

[카드 2. 리코더 잡는 법]
- 왼손이 위, 오른손이 아래! (가장 크게 강조)
- 왼손: 엄지는 뒷구멍(0번), 검지·중지·약지가 1·2·3번 구멍
- 오른손: 검지·중지·약지·새끼가 4·5·6·7번 구멍, 엄지는 뒤에서 받쳐 줍니다
일러스트: 기존 RecorderScene을 재사용해 손가락→구멍 연결선을 표시(왼손 파랑, 오른손 초록 계열 토큰 추가).

[카드 3. 입 모양]
- 리코더를 아랫입술에 얹고 윗입술로 가볍게 악기 취구를 덮습니다.
- 혀나 이가 취구에 닿지 않게 하며, 깊게 물지 않도록 주의합니다.
UI: "이렇게 해요 ⭕ / 주의해요 ⚠️" 두 칸 비교 박스.
⚠️ 예: 이로 깨물기 · 너무 깊게 물기 · 입술에 힘주기

[카드 4. 텅잉(Tonguing)]
- 텅잉이란, 취구로 들어가는 숨과 압력을 혀로 조절하는 것을 말합니다.
- '두(du)~' 하고 발음하듯 혀로 바람을 살짝 끊어 줍니다.
- 낮은음은 입안을 넓게 하고 '도(do)'나 '두(du)'에 가깝게 발음합니다.
인터랙션: [따라해 보기] 버튼 → "두 - 두 - 두 -" 글자가 80bpm 메트로놈처럼
하나씩 강조되는 반복 애니메이션. 시작/정지 가능, 화면 이탈 시 타이머 정리,
reduced-motion에서는 강조를 페이드로 대체.

[카드 5. 구멍을 막는 손가락 위치]
- 손가락 끝이 아니라 살집이 많은 부분(지문이 있는 볼록한 부분)으로 막아요.
- 지나치게 힘이 들어가지 않게 합니다.
- 손가락은 구멍으로부터 멀리 떨어지거나 너무 가깝게 두지 않습니다.
일러스트: 손가락 패드에 링 표시된 단순 SVG. 기존 finger-pad 컴포넌트 스타일 재사용.

[공통]
- 마지막 카드에 [운지 배우러 가기] 버튼 → 메인 학습 화면으로.
- 라우팅이 없다면 모드 상태로 전환하고, 있다면 /basics 경로.
- 각 카드 제목을 aria-level 2 헤딩으로, 슬라이드 전환을 라이브 영역으로 안내.

[테스트]
- 카드 5장 렌더·순환 컴포넌트 테스트
- 텅잉 메트로놈 시작→정지→언마운트 시 타이머 누수 없음(fake timers)
```

# U4 — 운지 체계 비교 뷰 (독일식 vs 바로크식)

## 목표

두 체계의 차이(사실상 '파')를 한 화면에서 나란히 이해시킨다.

## 실행 프롬프트

```text
설정의 운지 체계 항목 옆에 [비교해 보기] 버튼을 추가하고, 비교 뷰를 만들어.

[화면]
1. 좌우 2단(모바일 상하): 왼쪽 독일식, 오른쪽 바로크식.
   각 단은 축소된 RecorderScene(읽기 전용, 애니메이션 포함)을 렌더.
2. 상단에 음 선택(기본: 파). 파가 아닌 음을 고르면
   "이 음은 두 방식이 같아요" 배지를 크게 표시.
3. 파 선택 시 서로 다른 구멍(R4·R5·R6·R7 영역)에 차이 배지:
   독일식 쪽 "4번까지 차례로 막아요", 바로크식 쪽 "5번만 열고 6·7번을 막아요".
4. 하단 요약 카드(원문 유지):
   "독일식: 파를 쉽게 불 수 있어 학교에서 많이 써요.
    바로크식: 반음(올림·내림) 연주에 유리하고 세계적으로 널리 써요.
    내 리코더 뒷면에 B(바로크)나 G(독일) 표시를 확인해 보세요!"
5. [이 체계로 배우기] 버튼 → 해당 체계를 설정에 반영하고 메인으로 복귀.
   이때 현재 음이 파라면 원본 §4.5 규칙대로 전환 애니메이션을 보여준다.

[구현 원칙]
- RecorderScene을 복제하지 말고 props(system, note, readonly)로 재사용.
- 두 장면의 애니메이션 상태는 서로 독립(requestId 분리).

[테스트]
- 파에서 두 장면의 R5 상태가 서로 반대(독일식 closed 아님 → open/closed 정확히)임을 검증
- 솔에서 "두 방식이 같아요" 배지 표시 검증
```

## 완료 기준

- 파 기준 두 장면의 구멍 상태가 `FINGERINGS` 데이터와 정확히 일치하며, 체계 선택이 메인에 반영된다

# U5 — 퀴즈 모드 2종

## 목표

배운 운지를 게임으로 확인한다. 원본 §3.6의 언어 원칙("틀림" 대신 "한 번 더 확인해요")을 따른다.

## 실행 프롬프트

```text
features/quiz 모듈을 신설해 두 가지 퀴즈를 만들어.
공통: 한 라운드 10문제, 상단에 "문제 n/10"과 점수, 같은 음 연속 출제 금지,
현재 설정된 운지 체계를 따르며, 점수는 세션 메모리만 사용(로컬 저장 안 함).

[퀴즈 1. 운지 보고 계이름 맞히기]
1. RecorderScene(읽기 전용)에 무작위 음의 완성 운지를 표시. 계이름은 숨김.
2. 보기 4개: 정답 + 운지가 비슷한 오답 우선(diff 크기가 작은 음 순으로 선별).
3. 정답: 보기 버튼 성공 색 + 해당 음 500ms 재생 + "딩동댕!" → 1초 후 다음.
4. 오답: 누른 버튼을 흔들지 말고 테두리만 danger 색으로, 정답 버튼을 함께 표시,
   "한 번 더 확인해요. 정답은 ○○예요" 문구 + 정답 음 재생 → 다음.
5. 키보드로도 보기 선택 가능(1~4 또는 화살표+Enter).

[퀴즈 2. 계이름 보고 구멍 막기]
1. 문제: 계이름 크게 + [들어보기] 버튼.
2. 학생이 장면의 구멍(T0 포함 8개)을 직접 탭/클릭해 open↔closed 토글.
   이 모드에서만 Hole이 인터랙티브가 되며, 토글 시 손가락도 함께 움직인다.
3. [확인] → 정답과 비교: 맞은 구멍 성공 링, 다른 구멍은 안내 링 + 올바른 상태 표시.
4. 부분 점수: "8개 중 7개!" 형태. 만점이면 축하 사운드(짧은 상행 아르페지오).
5. 파 문제 특례: 현재 체계 기준으로 채점하되, 학생 답이 다른 체계의 파와
   정확히 일치하면 정답 처리하고 "그건 ○○식 파예요! 둘 다 맞아요" 안내.

[결과 화면]
- 90점 이상 "리코더 박사!", 70 이상 "잘했어요!", 그 외 "조금만 더 연습해요!"
- 헷갈린 음 목록 + [이 음 복습하기] → 메인에서 해당 음 선택 상태로 이동.
- [다시 도전] 버튼. 과도한 색종이 금지(원본 §15-12) — 결과 카드 페이드인 정도로.

[안정성·접근성]
- 문제 전환 중 연타로 이중 채점되지 않게 채점 후 입력 잠금.
- 퀴즈 중 탭 이탈→복귀 시 상태 유지 또는 안전한 초기화 중 택1 후 이유 보고.
- 구멍 토글은 키보드 포커스+Enter로도 가능, 상태 변화를 라이브 영역으로 안내.

[테스트]
- 오답 보기 생성이 정답을 중복 포함하지 않음
- 파 교차 체계 정답 인정 로직 단위 테스트
- E2E: 퀴즈2에서 구멍 3개 토글 → 확인 → 부분 점수 표시
```

## 완료 기준

- 두 퀴즈 모두 10문제 완주·채점·복습 이동이 정확하고, 채점이 `FINGERINGS`와 1:1 일치

# U6 — 연습 코스 확장 (자동 순서 연습 업그레이드)

## 목표

원본 Phase 9의 단일 시퀀스를 손 분리 학습 기반의 3개 코스로 확장한다.

## 실행 프롬프트

```text
자동 순서 연습을 코스 선택형으로 확장해.

[코스 3종 — 선택 카드]
1. 왼손 코스: 솔 → 라 → 시 → 높은 도 → 높은 레  (오른손은 계속 열림)
2. 오른손 코스: 도 → 레 → 미 → 파  (왼손은 계속 막음)
3. 한 옥타브 코스: 도 → … → 높은 레 (9음 전체)

[진행 방식 — 기존 자동 재생 로직 재사용]
1. 재생/일시정지/처음으로 + 음 간격 슬라이더(1.2~3.0초) 유지.
2. "수동 진행" 토글 추가: 켜면 자동 진행 대신 [다 했어요! 다음] 버튼으로 진행
   (학생이 실제 리코더로 따라 부는 수업 시나리오).
3. 각 음에서 buildInstruction의 diff 문장을 크게 표시하고,
   왼손 코스에는 "오른손은 그대로 쉬어요", 오른손 코스에는 "왼손은 그대로!"를 덧붙임.
4. 진행바 + "n/전체" 표시.

[완주 화면 — 절제된 보상]
- 별 3개가 순서대로 점등(각 300ms) + "○○ 코스 완주!" + 완주 코스 배지.
- 화면 전체 색종이 금지. 별 점등과 짧은 상행 아르페지오까지만.
- reduced-motion이면 별 3개 동시 표시.
- [다른 코스] [한 번 더] 버튼.

[테스트]
- 코스별 음 순서가 명세와 일치(단위 테스트)
- 자동 재생 중 다른 숫자 버튼 입력 시 시퀀스가 안전 중지(기존 규칙 회귀 확인)
```

## 완료 기준

- 3개 코스가 자동/수동 두 방식 모두로 완주 가능하고 기존 연타 안정성 테스트 통과
