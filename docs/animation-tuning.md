# 애니메이션 튜닝 가이드

현재 학습 무대는 **정렬된 완성 손 포즈 + 기능형 SVG 구멍 지도**로 구성됩니다. 조정 우선순위는 운지 정확성, 포즈 간 악기 정렬, 열린 손가락의 가독성, 전환의 자연스러움, 작은 화면 배치 순서입니다.

## 반드시 지킬 조건

1. 웹 포즈는 모두 `976×1360` 픽셀이고 같은 크롭 창을 사용합니다.
2. 리코더의 중심·길이·마우스피스와 벨 위치가 모든 파일에서 같아야 합니다.
3. 포즈 파일명과 음 매핑은 `poseAssets.ts` 한 곳에서 관리합니다.
4. 체계별 포즈가 다른 음은 `fa`, `faSharp`, `highFa` 세 개뿐입니다.
5. 파, 파♯, 높은 파는 각각 `*-baroque.png`와 `*-german.png`를 분리하고 나머지 14개 음은 포즈를 공유합니다.
6. 그림의 회색 손가락은 구멍에서 뗀 상태, 흰 손가락은 닿은 상태입니다.
7. SVG 지도의 정확한 상태는 PNG를 분석해서 만들지 않고 `FINGERING_STATES`의 `open`·`closed`·`half`·`partial`에서 직접 파생합니다.
8. `FINGERINGS`는 상태가 `open`이 아닌 구멍을 모은 접촉 애니메이션 호환 보기이며, 최종 반구멍·부분 막기 표현의 기준으로 사용하지 않습니다.
9. 17개 고유 음높이는 낮은 8개, 높은 5개, 반음 5개의 18개 선택 위치에 나타나며 C5는 두 분류가 공유합니다.
10. 빠른 입력 취소를 위해 상태 머신의 request ID와 포즈 캔버스의 generation/RAF 취소를 우회하지 않습니다.

## 포즈 캔버스 전환

파일: `src/features/recorder/components/RecorderPoseStage.tsx`

`RecorderPoseStage`는 최초 이미지를 그린 뒤 새 음이 들어올 때 현재 캔버스 자체를 스냅샷합니다. 그 스냅샷의 알파를 줄이고 목표 포즈의 알파를 늘려 전환합니다. 중간에 다른 음을 누르면 진행 중이던 RAF를 취소하고 현재 보이는 혼합 프레임에서 새 목표로 이어집니다.

| 설정 | 시간 | 용도 |
|---|---:|---|
| 보통 | 260ms | 빠르고 또렷한 기본 전환 |
| 느리게 | 720ms | 손 모양 차이를 자세히 관찰 |
| reduced motion | 0ms | 목표 포즈 즉시 표시 |

이징은 `easeOutQuint`입니다. 시간을 늘릴 때 상태 머신의 phase 전체 길이보다 지나치게 길어지지 않는지 확인합니다. 캔버스 전환 시간은 구멍 지도 phase 타이머와 별도이므로 하나를 바꿨다고 다른 값까지 기계적으로 같게 만들 필요는 없습니다.

20개 이미지는 처음 마운트될 때 모두 미리 읽습니다. 로드 실패 항목은 캐시에서 제거해 다음 시도에서 다시 읽을 수 있게 합니다. 캔버스를 사용할 수 없거나 아직 준비되지 않은 동안에는 `next/image` 폴백이 표시됩니다.

## 기능형 구멍 지도

파일: `src/features/recorder/components/RecorderFingeringMap.tsx`

지도는 `0 0 400 620` 좌표계를 사용하며 로컬 `MAP_HOLE_LAYOUT`이 위치를 결정합니다. 포즈 그림의 픽셀 좌표와 이 SVG 좌표를 일치시킬 필요는 없습니다. 지도는 손 모양의 보조 설명이므로 번호와 상태가 작은 화면에서도 분명한지가 더 중요합니다.

- T0은 뒤쪽 엄지 인셋 안에 둡니다.
- L1–L3은 왼손 위쪽 구멍입니다.
- R4–R7은 오른손 아래쪽 구멍입니다.
- R6/R7은 각각 두 원을 그리되 하나의 상태·라벨·전환을 공유합니다.
- 막힘은 짙은 채움, 열림은 흰 채움과 윤곽으로 구분합니다.
- `half`는 원의 절반을 채워 높은 미·파·솔의 T0 엄지 반구멍을 표시합니다.
- `partial`은 이중 구멍의 두 원 중 한쪽만 채워 도♯의 R7, 레♯·솔♯의 R6 부분 막기를 표시합니다.
- 떼기는 파란 점선 링, 막기는 주황 실선 링을 사용합니다.

`data-hole-id`, `data-hole-state`, `data-hole-transition`, `data-highlighted` 속성은 E2E와 컴포넌트 테스트가 사용하므로 이름을 바꿀 때 테스트도 함께 수정합니다.

상태 머신은 `FINGERINGS`의 접촉 배열로 손가락 떼기/누르기 순서를 계산합니다. 화면에 손가락이 닿아 있는 동안에는 같은 구멍의 목표 `FINGERING_STATES` 값을 합성하므로 `half`와 `partial`이 단순 `closed`로 사라지지 않습니다. 정면 포즈에서 뒷면 엄지나 이중 구멍 세부가 잘 보이지 않더라도 기능형 지도가 정확성 기준입니다.

## phase와 타이밍

파일: `src/features/recorder/animation/motionTimings.ts`

| 키 | 보통 | 느리게 | 역할 |
|---|---:|---:|---|
| `highlightRelease` | 60 | 150 | 뗄 구멍을 먼저 강조 |
| `release` | 160 | 300 | 논리 구멍을 열린 상태로 커밋 |
| `highlightPress` | 20 | 100 | 막을 구멍을 강조 |
| `press` | 180 | 400 | 논리 구멍을 막힌 상태로 커밋 |
| `contact` | 50 | 100 | 목표 접촉과 음 재생 유지 |
| `settle` | 50 | 150 | 강조를 안정 상태로 정리 |

```text
highlight-release → releasing → highlight-press → pressing → contact → settled
```

diff에 떼거나 막을 구멍이 없으면 해당 구간은 생략됩니다. 오디오는 최신 요청의 contact에서만 재생합니다. 새 타이머를 개별 컴포넌트에 추가하지 말고 중앙 스케줄러와 요청 취소 규약을 유지합니다.

## 단계별 보기

단계별 보기는 자동 타이머 대신 `STEP_PHASES`를 사용자가 진행합니다. 구멍 지도는 각 단계의 논리 상태와 강조를 보여 주고, 완성 손 포즈는 떼기·막기 안내가 끝나 실제 목표 자세를 보여 줄 시점에 전환합니다. 다음 음에 막을 구멍이 없는 경우에도 마지막 동작 뒤 목표 포즈와 지도가 같은지 확인합니다.

```text
음 이름 확인 → 열 구멍 강조 → 손가락 떼기 → 막을 구멍 강조
→ 손가락 누르기 → 완성 운지 확인 → 음 재생
```

`STEP_PHASES`나 표시 시점을 바꾸면 단계 번호, 안내 문구, 포즈 소스와 지도 상태를 한 테스트에서 함께 확인합니다.

## 조정 절차

1. 원본 AI에서 같은 크롭 창으로 필요한 포즈를 다시 렌더링합니다.
2. 20개 출력이 모두 `976×1360`인지, 악기 중심이 기존 파일과 같은지 이미지 편집기에서 겹쳐 봅니다.
3. `poseAssets.ts` 매핑과 파일명을 확인하고 `fa`, `faSharp`, `highFa`만 체계별 파일을 반환하는지 검사합니다.
4. 도→레, 미→파, 시→높은 도를 보통·느리게로 확인합니다.
5. 바로크↔독일식 파, 파♯, 높은 파에서 지도와 포즈가 동시에 체계별 목표로 바뀌는지 확인합니다.
6. `Shift+3` 높은 미, `Shift+4` 높은 파, `Shift+5` 높은 솔에서 T0이 `half`인지 확인합니다.
7. `Alt+1` 도♯, `Alt+2` 레♯, `Alt+4` 솔♯에서 해당 이중 구멍이 `partial`인지 확인합니다.
8. `1→8→4→5`와 분류 단축키를 빠르게 눌러 화면이 중간에 과거 포즈로 돌아가지 않는지 확인합니다.
9. 구멍 번호와 손가락 이름을 각각 켜고 모든 지도의 라벨을 확인합니다.
10. 360×800, 844×390, 데스크톱에서 세 분류 탭, 포즈·지도·변경 칩이 잘리지 않는지 확인합니다.

## 회귀 검수 체크리스트

- 도: 모든 손가락이 닿고 지도 0–7이 모두 막혔는가
- 레: 오른손 새끼손가락만 회색이고 R7만 열렸는가
- 미: R6/R7이 모두 열렸는가
- 바로크식 파: R5만 열리고 R6/R7은 막혔는가
- 독일식 파: R5/R6/R7이 모두 열렸는가
- 솔·라·시: 열린 손가락 수와 지도의 열린 구멍이 일치하는가
- 높은 도: T0과 L2만 막혔는가
- 도♯: R7이 `partial`, 레♯·솔♯: R6이 `partial`로 표시되는가
- 높은 레: T0이 열리고, 높은 미·파·솔: T0이 `half`로 표시되는가
- 파♯ 체계 변경 시 R7과 `fs4-baroque.png`/`fs4-german.png`가 함께 바뀌는가
- 높은 파 체계 변경 시 R6과 `f5-baroque.png`(PNG17)/`f5-german.png`(PNG16)가 함께 바뀌는가
- 파 체계 변경 시 포즈 파일과 지도 상태가 동시에 바뀌는가
- `fa`, `faSharp`, `highFa` 이외의 음은 체계 변경 뒤 같은 상태와 포즈를 유지하는가
- 낮은 8번과 높은 1번이 모두 C5와 `high-do.png`를 가리키는가
- 수정자 없음/Ctrl 1–8, Shift 1–5, Alt 1–5가 올바른 분류와 음을 선택하는가
- 복합 수정자, Meta, repeat, IME, AltGraph, Shift/Alt 6–8과 숫자패드가 무시되는가
- 느리게 보기에서 혼합 중 악기가 이중으로 흔들리지 않는가
- reduced motion에서 중간 페이드 없이 목표가 즉시 보이는가
- 빠른 연타 뒤 마지막 음에서만 contact와 소리가 발생하는가
